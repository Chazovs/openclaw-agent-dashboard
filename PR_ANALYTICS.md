# PR: Добавление панели аналитики производительности агентов

## Описание изменений

Этот PR добавляет комплексную систему аналитики производительности для OpenClaw Agent Dashboard. Новая фича предоставляет расширенный мониторинг, анализ трендов и рекомендации по оптимизации работы агентов.

### Ключевые возможности:
- 📊 **Мониторинг в реальном времени** статусов агентов и использования ресурсов
- 📈 **Исторические графики** изменения метрик за различные периоды
- ⚡ **Анализ эффективности** агентов (токены/время)
- 🚨 **Система оповещений** об аномалиях и проблемах
- 💡 **Интеллектуальные рекомендации** по оптимизации
- 📁 **Анализ распределения** по workspace

## Технические детали

### Архитектура:
```
Frontend (analytics.html) ↔ Backend API ↔ Analytics Engine ↔ Data Storage (JSON)
```

### API Endpoints:
- `GET /api/analytics/performance` - Текущие метрики
- `GET /api/analytics/history?hours=N` - Исторические данные
- `GET /api/analytics/workspaces` - Статистика по workspace
- `GET /api/analytics/summary` - Сводка производительности

### Метрики:
1. **Базовые**: Количество агентов, статусы, workspace
2. **Ресурсы**: Использование токенов (общее, среднее, максимальное)
3. **Эффективность**: Токены/час, тренды производительности
4. **Аномалии**: Оповещения трех уровней (error, warning, info)

## Как протестировать

1. **Клонируйте ветку**
2. **Установите зависимости**: `npm install`
3. **Запустите тесты**: `node test-analytics.js`
4. **Запустите сервер**: `node server-simple.js`
5. **Откройте в браузере**:
   - Главная: `http://localhost:3000/`
   - Аналитика: `http://localhost:3000/analytics.html`
6. **Проверьте API**:
   - `curl http://localhost:3000/api/analytics/performance`
   - `curl http://localhost:3000/api/analytics/history?hours=24`
