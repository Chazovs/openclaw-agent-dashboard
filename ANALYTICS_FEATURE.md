# 📊 Аналитическая панель OpenClaw Dashboard

## Обзор

Новая аналитическая панель предоставляет расширенную аналитику активности агентов, тренды использования токенов и системные метрики в реальном времени. Панель включает интерактивные графики, уведомления о критических событиях и детализированную статистику.

## Основные возможности

### 1. 📈 Интерактивные графики
- **График активности агентов**: Отображение количества работающих, простаивающих, спящих агентов и агентов с ошибками за последние 24 часа, 7 дней или 30 дней
- **График использования токенов**: 
  - Общее использование токенов
  - Среднее использование на агента
  - Топ-5 агентов по использованию токенов

### 2. ⚡ Сводные карточки
- **Статус системы**: Общий статус (нормальный/предупреждение/критический)
- **Активные агенты**: Текущее количество агентов с трендом за последний час
- **Использование токенов**: Общее количество использованных токенов
- **Работающие агенты**: Количество активных агентов

### 3. ⚠️ Система уведомлений
- **Критические уведомления**: Высокая загрузка CPU (>2.0), использование памяти (>85%)
- **Предупреждения**: Большинство агентов неактивны (>80%)
- **Ошибки**: Агенты в состоянии ошибки
- **Автоматическое обновление**: Уведомления обновляются в реальном времени

### 4. 💾 Системные метрики
- **Загрузка CPU**: Текущая загрузка, средняя за 1 минуту, количество ядер
- **Использование памяти**: Процент использования, свободно/использовано/всего
- **Время работы**: Аптайм системы

### 5. 🔄 Функциональность
- **Автообновление**: Данные обновляются каждые 30 секунд
- **Экспорт данных**: Возможность экспорта аналитики в JSON
- **Адаптивный дизайн**: Полная поддержка мобильных устройств
- **Темная тема**: Стильный дизайн в темных тонах

## Техническая архитектура

### Компоненты системы

#### 1. AnalyticsModule (analytics-module.js)
```javascript
// Основные функции:
- collectData(agents): Сбор и анализ данных
- analyzeAgents(agents): Анализ метрик агентов
- getSystemMetrics(): Получение системных метрик
- generateAlerts(agents, metrics): Генерация уведомлений
- getHourlyData(hours): Получение часовых данных
- getDailyData(days): Получение дневных данных
- getAnalyticsSummary(): Получение сводки
```

#### 2. API Endpoints
```
GET /api/analytics          # Полная аналитика
GET /api/analytics/summary  # Сводка аналитики
```

#### 3. WebSocket Events
```
analytics-update           # Обновление аналитики в реальном времени
```

#### 4. Frontend Components
- `analytics-panel.html`: Основной HTML-шаблон
- Встроенный JavaScript: Логика отображения и взаимодействия
- Chart.js: Визуализация графиков

## Установка и настройка

### Предварительные требования
- Node.js 14+
- OpenClaw Dashboard (базовая версия)
- Доступ к системным метрикам

### Установка
1. Скопируйте файлы в проект:
   ```
   analytics-module.js
   analytics-panel.html
   ```

2. Обновите `server.js`:
   ```javascript
   const AnalyticsModule = require('./analytics-module');
   const analyticsModule = new AnalyticsModule();
   
   // Добавьте новые API endpoints
   app.get('/api/analytics', ...);
   app.get('/api/analytics/summary', ...);
   ```

3. Обновите функцию `updateAgents()` для сбора аналитики

4. Добавьте ссылку на аналитическую панель в основной дашборд

### Конфигурация
```javascript
// Настройки в analytics-module.js
const ANALYTICS_DATA_PATH = './data/analytics.json';  // Путь к данным
const DATA_RETENTION_DAYS = 7;                        // Хранение данных (дни)
const ALERT_THRESHOLDS = {                            // Пороги уведомлений
  cpu: 2.0,      // Высокая загрузка CPU
  memory: 85,    // Высокое использование памяти
  inactive: 0.8  // Процент неактивных агентов
};
```

## Использование

### Доступ к панели
1. Откройте основной дашборд: `http://localhost:3000`
2. Нажмите кнопку "📊 Перейти к аналитике производительности"
3. Или перейдите напрямую: `http://localhost:3000/analytics-panel.html`

### Функции интерфейса

#### Фильтрация данных
- **Временные фильтры**: 24ч / 7д / 30д для графиков активности
- **Фильтры токенов**: Всего / В среднем / Топ-5 для графиков использования

#### Управление уведомлениями
- **Отметить как прочитанное**: Скрыть уведомление
- **Подробнее**: Просмотр деталей уведомления

#### Экспорт данных
- Нажмите кнопку "📥 Экспорт данных"
- Данные сохраняются в формате JSON
- Имя файла: `openclaw-analytics-YYYY-MM-DD.json`

## Мониторинг и обслуживание

### Хранение данных
- **Часовые данные**: Хранятся 7 дней
- **Дневные агрегации**: Хранятся 30 дней
- **Формат**: JSON в `./data/analytics.json`

### Очистка данных
- Автоматическая очистка старых данных
- Ручная очистка: Удалите `./data/analytics.json`

### Мониторинг производительности
- **Использование памяти**: ~10-50MB в зависимости от объема данных
- **Время обработки**: <100ms для сбора данных
- **Частота обновления**: Каждые 30 секунд

## Устранение неполадок

### Проблемы с данными
```
Проблема: Нет данных в графиках
Решение: Проверьте API endpoint /api/analytics
```

```
Проблема: Не обновляются уведомления
Решение: Проверьте WebSocket соединение
```

### Проблемы с производительностью
```
Проблема: Медленная загрузка панели
Решение: Увеличьте интервал обновления (setInterval)
```

```
Проблема: Высокое использование памяти
Решение: Уменьшите период хранения данных
```

### Проблемы с отображением
```
Проблема: Графики не отображаются
Решение: Проверьте подключение Chart.js
```

```
Проблема: Не работает адаптивный дизайн
Решение: Проверьте CSS медиа-запросы
```
