# Документация: Панель аналитики производительности агентов

## Обзор

Панель аналитики производительности - это новая фича для OpenClaw Agent Dashboard, которая предоставляет расширенный мониторинг и анализ производительности агентов. Система собирает метрики, анализирует тренды и предоставляет рекомендации по оптимизации.

## Функциональность

### 1. Основные возможности

- **Мониторинг в реальном времени**: Отслеживание статусов агентов и использования ресурсов
- **Исторический анализ**: Графики изменения метрик за выбранный период (1 час, 6 часов, 24 часа, 7 дней)
- **Анализ эффективности**: Расчет эффективности агентов (токены/время)
- **Система оповещений**: Автоматическое обнаружение аномалий
- **Рекомендации**: Интеллектуальные рекомендации по оптимизации

### 2. Метрики и показатели

#### Базовые метрики:
- **Количество агентов**: Общее число, по статусам, по workspace
- **Использование токенов**: Общее, среднее, максимальное
- **Эффективность**: Токены в час на агента

#### Производные метрики:
- **Тренды**: Изменение метрик во времени
- **Распределение**: По workspace, по статусам
- **Аномалии**: Отклонения от нормальных значений

### 3. Оповещения

Система генерирует оповещения трех уровней:

- **🔴 Error**: Критические проблемы (агенты с ошибками)
- **🟡 Warning**: Предупреждения (высокое использование токенов)
- **🔵 Info**: Информационные сообщения (длительное бездействие)

### API Endpoints

1. **GET /api/analytics/performance**
   - Возвращает текущие метрики производительности
   - Пример ответа: `{ success: true, analytics: {...} }`

2. **GET /api/analytics/history?hours=24**
   - Возвращает исторические данные для графиков
   - Поддерживает периоды: 1, 6, 24, 168 часов

3. **GET /api/analytics/workspaces**
   - Возвращает статистику по workspace
   - Включает распределение и топ агентов

4. **GET /api/analytics/summary**
   - Возвращает сводку производительности
   - Включает тренды и рекомендации

### Класс AnalyticsEngine

Основной класс для анализа данных:

```javascript
const analytics = new AnalyticsEngine();

// Анализ агентов
const metrics = analytics.analyzeAgents(agents);

// Получение исторических данных
const history = analytics.getHistoricalData(24);

// Статистика по workspace
const workspaceStats = analytics.getWorkspaceStats();

// Сводка производительности
const summary = analytics.getPerformanceSummary();
```

## Установка и настройка

### 1. Требования

- Node.js 14+
- Express.js
- Chart.js (включен через CDN)
- Существующий OpenClaw Agent Dashboard

### 2. Установка

1. Скопируйте файлы в директорию дашборда:
   ```
   analytics-engine.js
   public/analytics.html
   public/js/analytics/analytics.js
   public/css/analytics/analytics.css
   ```

2. Обновите `server-simple.js`:
   - Добавьте импорт AnalyticsEngine
   - Добавьте API endpoints
   - Инициализируйте движок аналитики

3. Добавьте ссылку на главную страницу:
   ```html
   <a href="/analytics.html">📊 Перейти к аналитике</a>
   ```

### 3. Конфигурация

Настройки по умолчанию находятся в классе `AnalyticsEngine`:

```javascript
this.alertThresholds = {
    highTokenUsage: 100000,    // Порог высокого использования токенов
    longInactivity: 3600000,   // 1 час бездействия
    errorRate: 0.1,            // 10% ошибок
    lowEfficiency: 0.5         // Низкая эффективность
};

this.maxHistorySize = 1000;    // Максимальное количество записей
```

## Использование

### 1. Веб-интерфейс

Откройте в браузере: `http://localhost:3000/analytics.html`

Интерфейс разделен на 5 секций:

1. **📈 Обзор**: Основные метрики и графики
2. **⚡ Производительность**: Топ агентов и эффективность
3. **📁 Workspace**: Распределение по workspace
4. **🚨 Оповещения**: Активные оповещения и история
5. **💡 Рекомендации**: Рекомендации по оптимизации

### 2. Фильтрация данных

- **Период времени**: 1 час, 6 часов, 24 часа, 7 дней
- **Автообновление**: Каждые 30 секунд
- **Экспорт данных**: В разработке

### 3. Интерпретация метрик

#### Эффективность агентов:
- **> 10000 т/ч**: Высокая эффективность
- **5000-10000 т/ч**: Средняя эффективность
- **< 5000 т/ч**: Низкая эффективность

#### Использование токенов:
- **< 50000**: Нормальное использование
- **50000-100000**: Повышенное использование
- **> 100000**: Высокое использование (оповещение)

## Производительность

### Оптимизации

1. **Ленивая загрузка**: Графики загружаются только при просмотре секции
2. **Кэширование**: Исторические данные кэшируются в памяти
3. **Пагинация**: Большие наборы данных разбиваются на страницы
4. **Дебаунсинг**: Частые обновления объединяются

### Ограничения

- **История**: Хранится 1000 последних записей
- **Память**: ~1MB на 1000 записей
- **Производительность**: Анализ 100 агентов занимает < 100ms

## Безопасность

### Меры безопасности

1. **Валидация входных данных**: Все API endpoints валидируют входные данные
2. **Ограничение запросов**: Rate limiting для API
3. **Санкционирование данных**: Чувствительные данные не отображаются
4. **XSS защита**: Экранирование HTML в интерфейсе

### Рекомендации по развертыванию

1. Используйте HTTPS в production
2. Настройте аутентификацию для доступа к аналитике
3. Регулярно обновляйте зависимости
4. Мониторьте использование памяти

## Устранение неполадок

### Распространенные проблемы

1. **Нет данных в графиках**
   - Проверьте API endpoint `/api/analytics/performance`
   - Убедитесь, что есть активные агенты
   - Проверьте консоль браузера на ошибки

2. **Оповещения не генерируются**
   - Проверьте пороги в `alertThresholds`
   - Убедитесь, что агенты имеют `realMetrics`
   - Проверьте логи сервера

3. **Медленная работа интерфейса**
   - Уменьшите период истории
   - Увеличьте интервал автообновления
   - Проверьте использование памяти
